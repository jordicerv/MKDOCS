    Llicenciament

    En aquest projecte, considerem essencial garantir que tot el programari utilitzat compleixi amb les condicions legals corresponents, i que l'empresa pugui operar amb seguretat i eficiència. Per això, s’ha fet una anàlisi del model de llicenciament que s’aplicarà tant als sistemes operatius com a les eines de producció i col·laboració que es faran servir.

    Tipus de llicenciament

    S’utilitzaran principalment dues tipologies de programari:

    • Programari lliure / codi obert (Open Source)
    • Programari privatiu / comercial

    Programari lliure

    El programari lliure, com Ubuntu Server o Nextcloud, permet l’accés al codi font i la seva modificació. Aquest tipus de programari és gratuït i legal d’utilitzar, modificar i distribuir. Les principals raons per triar-lo són:

    • Reducció de costos: No cal pagar llicències.
    • Flexibilitat: Possibilitat d’adaptar el programari a les necessitats específiques de l’empresa.
    • Suport comunitari: Hi ha una comunitat activa que facilita resolució de problemes i documentació.
    • Actualitzacions constants: Els projectes de codi obert solen tenir actualitzacions de seguretat freqüents.

    Exemples que s'utilitzaran: - Ubuntu Server com a sistema operatiu principal als servidors. - Nextcloud per al núvol privat i col·laboració documental. - OpenProject com a eina de gestió de projectes.

    Programari privatiu

    Malgrat que es prioritza el programari lliure, hi ha eines que, per funcionalitats o compatibilitat amb clients i col·laboradors externs, poden requerir l’ús de programari privatiu amb llicència comercial.

    Característiques del programari privatiu: - Limitació d’ús i modificació - Llicències específiques per ús (per dispositiu, per usuari, subscripcions, etc.) - Suport tècnic garantit pel fabricant

    Exemples possibles: - Microsoft 365: En cas que calgui compatibilitat ofimàtica completa amb clients. - Adobe Creative Cloud: Per a disseny gràfic i producció de continguts multimèdia (si cal). - Sistemes CRM/ERP específics amb funcionalitats avançades.

    Model de llicenciament adoptat

    Component Tipus de llicència Descripció / Observacions
    Ubuntu Server Lliure (GPL) Sistema operatiu per als servidors.
    Nextcloud Lliure (AGPL) Gestió de documents i núvol privat.
    OpenProject Lliure (GPLv3) Gestió de projectes.
    Microsoft 365 (opcional) Comercial Si cal compatibilitat amb documents Office.
    Adobe CC (opcional) Comercial Si es requereix per al departament de disseny.

    Gestió de llicències

    L'empresa serà responsable de:

    • Documentar totes les llicències utilitzades.
    • Vetllar pel compliment legal de cada eina instal·lada.
    • Gestionar renovacions de subscripcions en cas de programari privatiu.
    • Revisar de manera periòdica els termes de les llicències.

    Beneficis de la combinació lliure + privatiu

    Aquesta estratègia híbrida permet aprofitar el millor de cada món: - Estalvi econòmic gràcies a l’ús de solucions lliures. - Professionalitat i compatibilitat mitjançant llicències comercials quan cal. - Escalabilitat per créixer sense costos imprevistos. - Seguretat legal amb totes les eines degudament llicenciades.
